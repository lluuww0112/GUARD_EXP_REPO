DDPS : Dynamic Dense Patch Selection Keep the patches that matter.

DDPS is a train-free, plug-and-play visual token selection strategy that keeps only query-relevant patches before the visual sequence ever reaches the LLM. Instead of waiting for internal attention maps, DDPS reduces the sequence at the input stage and retains only the visual evidence that actually supports video reasoning.

Why This Matters

Why reduce visual tokens
before the LLM?

01

Most video patches are irrelevant

Only a small fraction of spatial patches actually contributes to answering a query.

02

Late pruning still pays early cost

Methods that rely on internal attention still spend expensive full-sequence computation before pruning.

03

Deployment should stay simple

Extra training and model surgery reduce the practicality of token reduction methods in real systems.

Core Observation

Query-relevant visual evidence
is usually local

For a query such as “What is the weather like outside?”, the model only needs the window region and the outdoor area, not every patch in the frame. The sofa, table, ceiling, and interior wall contribute little to the answer, while the visible sky, lighting, and scene outside the window contain most of the evidence needed for correct reasoning.

Method

How DDPS works

DDPS computes patch-text relevance before LLM inference,
combines it with frame-level importance and dynamically allocates a token budget across frames.

Pipeline Overview

Unlike methods that first run shallow LLM layers to obtain attention scores, DDPS reduces the visual sequence before the LLM input stage and therefore cuts early full-sequence computation directly. It first measures patch-text relevance in an external aligned vision-language space, then combines those patch scores with frame-level importance, and finally allocates the visual token budget dynamically across frames before the selected tokens are sent to the Video-LLM.

Video-LLM

Use MaskCLIP-based CLIP ViT-L/14 features to extract patch-level and frame-level aligned representations.

Score each patch with respect to the user query using cosine similarity in the shared vision-language space.

Distribute the visual budget across frames using frame relevance and the temperature parameter λ = 0.4.

Apply threshold τ = 0.02 and keep ratio ρ to keep only useful patches before sending them to the Video-LLM.

Patch-Level

Select directly from aligned visual evidence

DDPS does not depend on intermediate LLM attention and instead uses an external alignment model to identify useful patches.

Frame-Level

Allocate tokens where the query needs them

Frames with higher query relevance receive more of the visual budget instead of enforcing uniform allocation.

Backbone-Friendly

Apply without retraining the Video-LLM

The method lives in the input pipeline, which makes it suitable for plug-and-play deployment on existing backbones.

Qwen3-VL-Specific DDPS Setup

Qwen3-VL internally merges adjacent frames, so sparse uniform sampling can lose temporal evidence when two different sampled frames are compressed together.

To avoid that mismatch, DDPS duplicates each sampled frame once so that every frame is merged with an identical copy of itself rather than a different neighbor.

Qwen-family models also merge each adjacent 2 × 2 patch group into one visual token. DDPS therefore averages dense patch-text scores over each merge unit and aligns the relevance map to the exact token grid consumed by Qwen3-VL.

With this setup, DDPS preserves the temporal information protected by frame duplication while still pruning the merged visual sequence before the Video-LLM processes it.
